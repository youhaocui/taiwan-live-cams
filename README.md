## 項目簡介

本專案是一個專為台灣即時影像直播源設計的優化網路應用程式。它提供一個同步、穩定的多分割監看平台，讓使用者能高效地一次觀看 YouTube 上台灣各地的即時畫面。

### 🌟 核心亮點：即時同步零延遲

本工具的核心優勢在於其**四分割畫面**設計，以及專為解決 YouTube 直播延遲而開發的 **Live Edge 即時同步穩定機制**。此機制能確保畫面強制跳轉至直播前緣 (Live Edge)，提供**最接近實況的監看體驗**。

---

## 🚀 核心功能與特色

* **四分割同步監看：** 在單一介面上同步顯示四個不同的 YouTube 即時直播畫面。
* **⚡ Live Edge 即時同步：** 內建強制同步機制，透過週期性檢查，將影片強制跳轉至**直播前緣 (Live Edge)**，有效解決 YouTube 延遲，保持畫面極度即時。
* **🎶 獨立背景音樂播放器：** 獨立於直播畫面的背景音樂功能，不影響監看。
    * 支援輸入單曲 ID 或播放清單 ID。
    * 提供列表循環 (List Loop) 和隨機播放 (Shuffle) 模式。
    * 播放器預設為靜音且隱藏。
* **地點快速切換：** 底部選單依地區分類，可快速將選定的直播源載入至四個畫格中的任一位置。
* **響應式控制：** 提供一鍵**全螢幕切換**、標籤**隱藏/顯示**、全體**靜音/解除靜音**等快速操作按鈕。

### 🌐 語言支援

本專案提供多語言介面，使用者可隨時切換，以滿足不同語系的需求。

* **目前支援：** **繁體中文 (zh-TW)** 與 **英文 (en)**。

## 🛠️ 開發與部署指南

### 部署環境要求

本專案必須透過 Web 伺服器環境運行。由於使用了 YouTube Iframe API，瀏覽器安全限制禁止直接在本地電腦上開啟 `.html` 檔案。

### Live Edge 穩定機制運作原理

穩定性仰賴 JavaScript 中的 `checkStreamDelayLoop` 函式。此函式會定期計算直播延遲，一旦延遲超過 **30 秒**，即會自動執行 `player.seekTo(player.getDuration(), true)` 進行畫面同步。
