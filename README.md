## 項目簡介

本專案是一個優化且高效的網路應用程式，旨在提供一個同步、穩定的平台，用於監看 YouTube 上台灣各地的即時影像直播源。

**核心優勢** 在於其四分割畫面設計，以及專為解決 YouTube 直播畫面延遲問題而開發的 **Live Edge 即時同步**穩定機制，確保使用者能夠獲得最接近實況的監看體驗。

---

## 🚀 核心功能與特色

* **四分割即時監看：** 同步顯示四個不同的 YouTube 直播畫面。
* **⚡ Live Edge 穩定：** 內建強制同步機制，透過週期性檢查，將影片強制跳轉至直播前緣 (Live Edge)，有效保持畫面即時性。
* **🎶 背景音樂播放器：** 獨立於直播畫面的背景音樂功能。
    * 可輸入單曲 ID 或播放清單 ID。
    * 支援列表循環（List Loop）和隨機播放（Shuffle）模式。
    * 播放器預設靜音且隱藏。
* **地點快速切換：** 底部選單依地區分類，可快速將選定的直播源載入至四個畫面中的任一位置。
* **響應式控制：** 提供一鍵全螢幕切換、標籤隱藏、全體靜音/解除靜音等快速控制按鈕。

## 🛠️ 開發與部署指南

### 部署環境要求

本專案必須透過 Web 伺服器環境（例如：GitHub Pages, Netlify）運行。由於使用了 YouTube Iframe API，瀏覽器安全限制禁止直接在本地電腦上開啟 `.html` 檔案。


### 2. Live Edge 穩定機制

穩定性仰賴 JavaScript 中的 `checkStreamDelayLoop` 函式。此函式會定期計算直播延遲，一旦延遲超過 **30 秒**，即會自動執行 `player.seekTo(player.getDuration(), true)` 進行畫面同步。
